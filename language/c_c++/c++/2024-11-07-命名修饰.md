## 定义

C++为支持函数重载, 特有命名修饰(Name Mangling)机制

编译阶段编译器会通过添加参数类型、参数个数等额外信息重命名函数, 生成唯一符号名, 以区分具有相同名字但不同参数列表的函数

链接阶段链接器需要依赖符号名来解析不同目标文件和库文件中符号引用, 正确区分和链接函数

- 示例, 两个同名函数Sum经过`Name Mangling`后生成唯一修饰名

```mermaid

    A-->C-->A1
    B-->C-->B1
```

## 使用

函数和变量在本质上都是地址助记符, 在链接过程中称为`符号`(Symbol)

### C语言

C语言不支持重载, 故无命名修饰机制, 每个函数都有唯一未修饰名称，链接器可直接使用名称解析符号

```c
// CModule.h
#include<stdio.h>
int AddNum(int x, int y);
void PrintValue(double num);
```

```c
// CModule.c
#include "CModule.h"
int AddNum(int x, int y) {
    return x + y;
}
void PrintValue(double num) {
    printf("res = %f\n", num);
}
```

编译为目标文件

```sh
gcc CModule.c -c -o CModule.o
```

- 符号表

```sh

```

int Add(int x, int y) {

- 原因分析

Main.cpp 预处理时, 内容展开

```diff
+ #include<stdio.h>
+ int Add(int x, int y);
+ double GetSquareArea(double length);
#include<iostream>
int main() {
    int x = 1;
    int y = 2;
    int res = Add(x, y);
    double length = 3.74;
    double area = GetSquareArea(length);
    std::cout << "Add = " << res << std::endl;
    std::cout << "SquareArea = " << area << std::endl;
    return 0;
}
```

编译时C++编译器对Main.cpp中原本C语言函数`Add`、`GetSquareArea`使用命名修饰机制, 生成全新函数名

Math.o因为是C编译器编译, 没有经过命令修饰, 函数名未改变

链接时Main.o按照`_Z3Addii`名字到各模块查找函数引用, 结果Math.o里函数名称是`Add`, 自然出现函数未定义错误, 需通过`extern "C"`处理

## extern "C"

### 定义

C++各编译器中提供 `extern "C"`/ `extern "C" {}`语句, 表示后续或范围内函数强制按C语言风格编译, 暂时禁用命名修饰机制

通常用于C++代码中调用C语言代码

### 语法

- 示例, 函数声明

表示Func函数按照C语言规则编译

```c
extern "C" void Func();
```

- 示例, 代码块作用

表示花括号内所有声明都按照C语言规则处理

```c
extern "C" {
    void Func1();
    void Func2();
}
```

- 示例, 仅C++编译时使用

预处理宏`__cplusplus`仅在C++编译器中被定义

下面代码表示若使用C++编译器编译, 则对花括号内函数按C语言规则出

```c
#if __cplusplus
extern "C" {
#endif
    void Func1();
    void Func2();
#if __cplusplus
}
#endif
```

### 应用

#### C++调用C语言代码

例如对于上述未定义错误, 有两种方式修改

- 方式1, 仅修改Math.h

表示若C++引用Math.h时, 对声明函数添加`extern "C"`, 按C语言规则编译

```c++
#include<stdio.h>

#if __cplusplus
extern "C" {
#endif
    int Add(int x, int y);
    double GetSquareArea(double length);
#if __cplusplus
}
#endif
```

Main.cpp预处理时展开

```c++
// Math.h
#include<stdio.h>

#if __cplusplus
extern "C" {
#endif
    int Add(int x, int y);
    double GetSquareArea(double length);
#if __cplusplus
}
#endif

#include<iostream>
int main() {
    int x = 1;
    int y = 2;
    int res = Add(x, y);
    double length = 3.74;
    double area = GetSquareArea(length);
    std::cout << "Add = " << res << std::endl;
    std::cout << "SquareArea = " << area << std::endl;
    return 0;
}
```

- 方式2, 仅修改Main.cpp

对于引用C语言头文件直接使用`extern "C" {}`处理

```c++
extern "C" {
    #include "Math.h"
}

#include<iostream>
int main() {
    int x = 1;
    int y = 2;
    int res = Add(x, y);
    double length = 3.74;
    double area = GetSquareArea(length);
    std::cout << "Add = " << res << std::endl;
    std::cout << "SquareArea = " << area << std::endl;
    return 0;
}
```

Main.cpp预处理时展开

```c++
extern "C" {
    #include<stdio.h>

    int Add(int x, int y);
    double GetSquareArea(double length);
}

#include<iostream>
int main() {
    int x = 1;
    int y = 2;
    int res = Add(x, y);
    double length = 3.74;
    double area = GetSquareArea(length);
    std::cout << "Add = " << res << std::endl;
    std::cout << "SquareArea = " << area << std::endl;
    return 0;
}
```

两种方式由于`extern "C"`存在, 编译Main.cpp时两个C语言函数生成符号不受命名修饰影响, 保持原名称, 和Math.o中符号一致
